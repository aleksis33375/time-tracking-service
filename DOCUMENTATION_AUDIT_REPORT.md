#### 🔴 ОШИБКА #2: Отсутствие информации о root package.json

**Строки:** 60-76 (раздел 1.2)

**Проблема:** В документации не упоминается, что в корне проекта находится `package.json` с зависимостями `sharp` и `tesseract.js`, необходимыми для webhook функции.

**Текущее содержание:**
```
Vercel автоматически:
- Установит зависимости из `bot/package.json`
- Соберёт webhook функцию
```

**Проблема:** 
- `bot/package.json` — это для регистрации webhook'а (register-webhook.js)
- Основные зависимости (sharp, tesseract.js) находятся в корневом `package.json`
- Это может привести к confusion при деплое

**Решение:** Уточнить в документации:

```
Vercel автоматически:
- Установит зависимости из корневого `package.json` (sharp, tesseract.js)
- Установит зависимости из `bot/package.json` для webhook registration
- Соберёт webhook функцию `api/webhook.js`
```

**Статус:** 🔴 Требует исправления

#### 🟠 ОШИБКА #3: Неточное описание Framework

**Строки:** 58-59

**Текущее содержание:**
```
Framework: **None** (не выбирайте Next.js)
Root Directory: **.** (корень репозитория)
```

**Проблема:** 
- Документация говорит "не выбирайте Next.js", но не объясняет, что будет выбрано
- В vercel.json указано только `outputDirectory: "."`, нет явного `framework`
- Это может привести к вопросам при настройке

**Решение:**
```

Root Directory: **.** (корень репозитория)

Vercel обнаружит:
- Серверную функцию: api/webhook.js
- Статику: dashboard/index.html
- Package.json с зависимостями
```

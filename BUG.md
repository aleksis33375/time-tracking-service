### BUG-001: Vercel timeout убивает OCR — события теряются навсегда

- **Файл:** (отсутствует) `/vercel.json`
- **Приоритет:** 🔴 КРИТИЧЕСКИЙ
- **Статус:** ✅ Исправлено 2026-05-08
- **Описание:** OCR занимает 20–30 сек, Hobby-план Vercel = 10 сек. Функция убивается до `insertEvent()`, Telegram получил `200 OK` → событие потеряно **безвозвратно**.
- **Решение:** Создать `/vercel.json`:
```json
{
  "functions": {
    "api/webhook.js": { "maxDuration": 60 },
    "api/sign-photo.js": { "maxDuration": 10 }
  }
}
```

### BUG-002: CHECK constraint не включает 'duplicate' — бесконечный цикл

- **Файл:** `supabase/events.sql:10`
- **Приоритет:** 🔴 КРИТИЧЕСКИЙ
- **Статус:** ✅ Исправлено 2026-05-08
- **Описание:** AI-worker пытается сохранить `status='duplicate'` → PostgreSQL отклоняет → событие застревает в `processing` 15 мин → восстанавливается в `pending` → снова ошибка → **бесконечный цикл**.
- **Решение (Supabase SQL Editor):**
```sql
ALTER TABLE public.events
  DROP CONSTRAINT IF EXISTS events_status_check,
  ADD CONSTRAINT events_status_check
    CHECK (status IN ('pending','processing','done','needs_review','duplicate','rejected'));
```

### BUG-037: Token в localStorage — XSS уязвив при наличии XSS

- **Описание:** GitHub Actions = Ubuntu, не влияет на продакшн. Функция `with_timeout` сделана кросс-платформенной: на Linux используется SIGALRM, на Windows — threading-фолбэк с `Thread.join(timeout)`.

## 🟠 ВЫСОКИЙ — BUG-042: needs_review без флага — причина не видна ревьюеру

- **Файл:** `ai-worker/process_events.py:928-929`
- **Приоритет:** 🟠 ВЫСОКИЙ
- **Статус:** ✅ Исправлено 2026-05-29
- **Описание:** Когда `is_incomplete_day()` возвращает `True` (arrival без парного departure), код устанавливает `go_review = True`, но **не добавляет никакого флага** в `fraud_flags`. Событие переходит в `needs_review` с пустым массивом `[]` — ревьюер видит карточку без объяснения причины. Аналогично когда `event_type is None` (строка 865) — `go_review = True` без флага.
- **Масштаб:** Минимум 20 событий в needs_review с `fraud_flags=[]` прямо сейчас.
- **Решение:** Добавить флаг при установке `go_review = True`:
```python
# Строка 928–929 — неполный день
if is_incomplete_day(employee, event_type, hours):
    if "incomplete_day" not in fraud_flags:
        fraud_flags.append("incomplete_day")
    go_review = True

# Строка 864–865 — неизвестный тип
if event_type is None:
    if "unknown_event_type" not in fraud_flags:
        fraud_flags.append("unknown_event_type")
    go_review = True
```

### TODO-007: Обработка сетевых ошибок в Dashboard

**Файл:** `dashboard/index.html:1090-1102` (функция `supabaseFetch`)  
**Время:** 30 минут  
**Сложность:** Средняя

**Текущее состояние:**
```javascript
async function supabaseFetch(path, options = {}) {
  const res = await fetch(...);  // ← нет try-catch, нет timeout
  return res.json();
}
```

**Проблема:** При сетевых ошибках Promise отклоняется, вызывающая функция падает.

**Решение:**
1. Добавить try-catch вокруг fetch
2. Добавить AbortSignal.timeout(10000) для 10-секундного таймаута
3. Проверить res.ok перед парсингом JSON
4. Логировать ошибки в консоль

**Тест:**
- Отключить интернет → Dashboard должен показать ошибку, а не упасть
- Медленный интернет (throttle) → Должен timeout через 10 сек
- 500 ошибка от сервера → Должна обработаться корректно

**Чек-лист:**
- [ ] Добавить try-catch
- [ ] Добавить timeout 10 сек
- [ ] Проверить res.ok
- [ ] Тестировать все сценарии
- [ ] Обновить вызывающие функции (если нужно)

### TODO-012: Валидация при импорте CSV

**Файл:** `dashboard/index.html:2195-2280` (функция `importPasteInput`)  
**Время:** 50 минут  
**Сложность:** Высокая

**Текущее состояние:**
```javascript
const rows = csv.split('\n').map(line => line.split(','));
rows.forEach(row => {
  // ...insert in DB
  // Нет валидации!
});
```

**Проблема:** Импортируются некорректные данные без проверки.

**Решение:**
1. Создать функцию `validateEmployeeRow(row)` 
2. Проверить каждое поле:
   - display_name: не пусто, не более 100 символов
   - position: не пусто, не более 50 символов
   - team: не пусто, существует в списке команд
   - daily_rate: число, > 0
3. Проверить нет дубликатов имён
4. Если ошибки — показать список ошибок
5. Если всё ОК — импортировать

**Пример валидации:**
```javascript
function validateEmployeeRow(row, rowNum) {
  if (!row[0] || !row[0].trim()) 
    return `Строка ${rowNum}: имя обязательно`;
  if (row[0].length > 100) 
    return `Строка ${rowNum}: имя слишком длинное`;
  const rate = parseFloat(row[2]);
  if (isNaN(rate) || rate <= 0) 
    return `Строка ${rowNum}: ставка должна быть положительным числом`;
  return null;  // OK
}
```

**Тест:**
- Импортировать с пустым именем → Ошибка на строке N
- Импортировать с невалидной ставкой → Ошибка
- Импортировать дубликаты → Ошибка
- Импортировать валидные данные → Успех

**Чек-лист:**
- [ ] Создать validateEmployeeRow()
- [ ] Проверить все поля
- [ ] Обнаружить дубликаты
- [ ] Показать список ошибок
- [ ] Не импортировать если ошибки

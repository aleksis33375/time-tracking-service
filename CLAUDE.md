# CLAUDE.md — AI Attendance System

## Общие правила процесса

1. Перед созданием нового файла — сначала расскажи, что планируешь сделать
2. Все предложения и варианты решения проблем делать исходя из наличия только трёх ресурсов: **GitHub · Supabase · Vercel**. Сторонние платные сервисы, VPS и внешние хостинги не предлагать.

## Структура проекта

```
/dashboard/index.html  ← весь фронтенд (Dashboard)
```

---

## Dashboard (frontend)

3. Один `index.html` — стили внутри `<style>`, скрипты внутри `<script>`
4. SPA-подход — секции показываются/скрываются через JS, без перезагрузки страницы
5. Без внешних CDN, Google Fonts, иконочных библиотек
6. Иконки: только inline SVG

### Дизайн
- Светлый фон (`#FFFFFF`), серые границы (`#E5E7EB`)
- Акцентный цвет: синий `#2563EB`
- Карточки: `border-radius: 8px`, лёгкая тень
- Адаптивная вёрстка (mobile-first, breakpoint: 768px)

### Стиль текста
- Шрифт: `system-ui`, без внешних подключений
- Body: 14–16px, `line-height: 1.5`
- Заголовки: жирные, 20–32px
- Тон: деловой, короткий, без воды

---

## Python-компоненты (bot, ai-worker)

9. Python: версия 3.10+
10. Использовать стандартные pip-пакеты (`face_recognition`, `python-telegram-bot`, `supabase-py` и др.)
11. Каждый компонент — в своей папке со своим `requirements.txt`
12. Версии пакетов фиксировать строго (`==`, не `~=`)

---

## Безопасность

13. Все ключи и токены — только в `.env`
14. `.env` добавить в `.gitignore`, в код не коммитить

# CLAUDE.md — AI Attendance System

## Общие правила процесса

1. Перед созданием нового файла — сначала расскажи, что планируешь сделать
2. Прежде чем начать работу, скажи, как будешь проверять результат. Если способа проверки нет — сообщи об этом и попроси уточнить задачу.
3. Все предложения делать исходя из трёх ресурсов: **GitHub · Supabase · Vercel**. Сторонние платные сервисы, VPS и внешние хостинги не предлагать.
4. **Любой найденный баг немедленно документировать в `BUG.md`** — с номером (BUG-NNN), файлом, приоритетом (🔴/🟠/🟡), статусом, описанием и предложенным решением. Не ждать явной просьбы.

## Структура проекта

```
/api/webhook.js        ← Telegram webhook (Vercel Serverless, Node.js)
                         Тонкий обработчик: парсинг подписи + сохранение фото.
                         OCR не используется — см. project.md.
/dashboard/index.html  ← Frontend (Dashboard)
/ai-worker/            ← Face Recognition + расчёт часов
                         (Python, запускается через GitHub Actions каждые 5 мин)
/bot/                  ← Утилиты: register-webhook.js и др. (Node.js)
/supabase/             ← SQL-схемы (Edge Functions не используются)
```

---

## Dashboard (frontend)

4. Один `index.html` — стили внутри `<style>`, скрипты внутри `<script>`
5. SPA-подход — секции показываются/скрываются через JS, без перезагрузки страницы
6. Без внешних CDN, Google Fonts, иконочных библиотек
7. Иконки: только inline SVG

### Дизайн
- Светлый фон (`#FFFFFF`), серые границы (`#E5E7EB`)
- Акцентный цвет: синий `#2563EB`
- Карточки: `border-radius: 8px`, лёгкая тень
- Адаптивная вёрстка (mobile-first, breakpoint: 768px)

### Стиль текста
- Шрифт: `system-ui`, без внешних подключений
- Body: 14–16px, `line-height: 1.5`
- Заголовки: жирные, 20–32px
- Тон: деловой, короткий, без воды

---

## Python-компоненты (ai-worker)

8. Python: версия 3.10+
9. Использовать стандартные pip-пакеты (`face_recognition`, `supabase-py` и др.)
10. Каждый компонент — в своей папке со своим `requirements.txt`
11. Версии пакетов фиксировать строго (`==`, не `~=`)

---

## Безопасность

12. Все ключи и токены — только в `.env`
13. `.env` добавить в `.gitignore`, в код не коммитить
